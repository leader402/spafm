# 论文要点解读

> Liu T, Hao M, Liu X, Zhao H. *A perspective on developing foundation models for analyzing spatial transcriptomic data*. **Quantitative Biology**, 2025;e70010. <https://doi.org/10.1002/qub2.70010>
>
> 原文 PDF：[paper/](../../paper/)

---

## 1. 核心问题

> "我们到底需不需要一个用于空间转录组学（ST）分析的基础模型（FM）？"

作者立场：**需要**，但 FM 必须解决简单方法解决不了的问题，否则不应被冠以 FM 之名。

---

## 2. ST 与 scRNA-seq 的关键差异

| 维度 | scRNA-seq | ST |
|---|---|---|
| 分辨率 | 单细胞 | spot / 亚细胞 / 单细胞（依平台） |
| 噪声来源 | 测序、dropout | + cell segmentation、niche 标注、未测基因 |
| 空间信息 | 无 | **必须建模** |
| 数据成本 | 较低 | 高 → 训练语料质量要求更高 |
| 序列结构 | 无显式序列 | 同样无显式序列结构 |

**结论**：scFM 的预处理与训练 paradigm 不能直接迁移。

---

## 3. 两种 FM 范式（论文 Figure 1）

### A. Seq-based FM —— 数据驱动

| 模型 | 核心思想 |
|---|---|
| **NicheCompass** | 图自编码器 + cell-cell interaction 作为 covariate embedding |
| **Novae** | 图感知编码器 + 对比学习 |
| **Nicheformer** | 把空间坐标作为独立 modality token |
| **STFormer** | 联合 niche-level + gene expression，**gene-gene interaction recovery** 作为新预训练目标 |
| **SpaFormer** | Transformer autoencoder，专做插补 |
| **CellPLM** | 把 cell 当 token；ST 作为 scRNA 的数据增强 |

### B. Knowledge-based FM —— 知识驱动

| 模型 | 核心思想 |
|---|---|
| **QuST-LLM** | 图像 + pathway 文本结合 |
| **Geneverse** | 多模态 LMM 微调识别 marker |

### C. 二者融合（趋势）

- **spEMO**：病理 FM 编码 H&E 图像 embedding 联合表达 embedding
- **scGPT-spatial**：在 scGPT 上 continual pretraining

---

## 4. ST FM 应承担的"高级任务"

1. **统一并自动选择最优预处理流程**（QC、normalization、integration、clustering、annotation）—— 解决主观性、保证可重复性
2. **作为先验知识增强下游模型**（参考 GenePT / scELMo / scFoundation 提供的 cell/gene/text embedding）
3. **驱动新生物发现**：novel cell type 发现、空间扰动效应预测、空间发育、肿瘤微环境

### 评测指标层级

| 任务 | 指标 |
|---|---|
| 注释 | F1 |
| Niche clustering | AMI、ASW |
| 基因插补 | Pearson、Cosine |
| Deconvolution | Pearson、SSIM、RMSE、JS |

---

## 5. 主要挑战

1. **高质量、多样化语料的构建与存储**
2. **缺乏针对"无显式序列结构"的合理预训练任务**
   作者建议从 **时空轨迹（spatio-temporal trajectory）** 角度设计新目标
3. **Benchmark 框架要兼顾 low-level 与 high-level 任务**
4. **计算成本**：建议开源不同规模模型 + 在线 demo

---

## 6. 警示：scFM 的现状

论文引用 [13–16]：

> 当前 scFM 性能并**不**与算力消耗成正比，甚至**不如线性方法**。

➡ ST FM 必须证明自己的"必要性"，否则不应被冠以 FM 之名。

---

## 7. 对 SpaFM 项目的指导意义

| 论文观点 | SpaFM 应对策略 |
|---|---|
| ST 噪声大、需空间上下文 | Stage 1 统一预处理 + 显式空间图建模 |
| 序列结构缺失 → 需要新预训练目标 | Stage 4 设计 MGM + Niche CL + ST-Trajectory + GG-Recovery |
| seq + knowledge 融合是趋势 | Stage 5 集成 LLM gene embedding + 病理图像 FM |
| 必须打过线性 baseline | Stage 7 SpaFM-Bench 强制对比 PCA/Harmony/线性回归 |
| 开源不同规模 + Demo | Stage 8 / Stage 10 发布 S/B/L 三档 + Gradio Demo |
| 时空轨迹是新角度 | Stage 4 引入连续切片 / 发育时序作为预训练任务 |
